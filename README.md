# Brave Love

[![Version](https://img.shields.io/badge/version-1.0.6-ff5162.svg)](https://github.com/unclecat/WP_Brave/releases)
[![WordPress](https://img.shields.io/badge/WordPress-6.0%2B-21759b.svg)](https://wordpress.org/)
[![PHP](https://img.shields.io/badge/PHP-7.4%2B-777bb4.svg)](https://www.php.net/)
[![License](https://img.shields.io/badge/license-GPL%20v2%20or%20later-2ea44f.svg)](https://www.gnu.org/licenses/gpl-2.0.html)

一款为情侣记录而生的 WordPress 主题，用更统一的杂志感排版来承载你们的故事、照片、清单与日常。

![Brave Love Theme Preview](./screenshot.png)

当前稳定版：`v1.0.6`

完整用户手册：[`docs/USER-GUIDE.md`](./docs/USER-GUIDE.md)

Brave Love 不是单独的某一个页面模板，而是一套完整的情侣纪念站主题：

- `首页`：双计时器、今日天气、特别的日子、六大入口卡片
- `关于我们`：故事时间线，适合整理从相遇到现在的重要节点
- `点点滴滴`：按年份归档的时间轴文章
- `恋爱清单`：计划 / 已完成事项管理
- `甜蜜相册`：自动从点滴中汇总照片
- `随笔说说`：轻量短内容与前台快捷发布
- `祝福留言`：基于 WordPress 评论系统的公开留言板

如果你希望搭一个“能长期维护”的情侣网站，而不是只做一个静态落地页，这个主题就是为这个场景准备的。

## 核心能力

| 模块 | 作用 | 后台维护位置 |
| --- | --- | --- |
| 首页 | 展示恋爱时长、特别日子、天气与站点入口 | `外观 -> 自定义`、`设置 -> 天气城市`、`设置 -> 纪念日` |
| 关于我们 | 按时间顺序展示故事节点 | `关于我们` 文章类型 + 关于我们页面 |
| 点点滴滴 | 发布长文记录、地点、心情与摘要 | `点点滴滴` 文章类型 |
| 恋爱清单 | 记录想做的事和已完成事项 | `恋爱清单` 文章类型 + `清单分类` |
| 甜蜜相册 | 自动聚合点滴中的特色图与正文图片 | 无需单独录入，来源于 `点点滴滴` |
| 随笔说说 | 记录日常短句、心情和思念度 | `随笔说说` 文章类型，或前台快捷发布 |
| 祝福留言 | 收集亲友留言并审核显示 | `祝福留言` 页面评论 |

## 环境要求

- WordPress `6.0+`
- PHP `7.4+`
- 推荐开启固定链接
- 推荐使用 HTTPS 站点地址

## 安装方式

### 方式一：从 Release 安装
1. 在 [Releases](https://github.com/unclecat/WP_Brave/releases) 下载 `brave-love.zip`
2. 进入 WordPress 后台 `外观 -> 主题 -> 添加主题 -> 上传主题`
3. 上传并启用 `Brave Love`
4. 进入 `设置 -> 固定链接`，直接点击一次“保存更改”刷新重写规则

### 方式二：手动安装
1. 将主题目录放入 `wp-content/themes/`
2. 进入后台启用 `Brave Love`
3. 同样执行一次 `设置 -> 固定链接 -> 保存更改`

## 建站前先做这 6 步

1. 创建页面并分配模板：`首页 / 关于我们 / 点点滴滴 / 甜蜜相册 / 随笔说说 / 祝福留言`
2. 进入 `设置 -> 阅读`，把首页设为静态首页
3. 进入 `外观 -> 自定义 -> Brave 主题设置`，配置恋爱起始时间、头像、页面链接、页脚导航
4. 进入 `设置 -> 天气城市`，添加你关心的城市坐标
5. 进入 `设置 -> 纪念日`，录入重要日子
6. 开始添加 `点点滴滴 / 关于我们 / 恋爱清单 / 随笔说说` 数据

## 必建页面与模板对应关系

| 前台页面 | 是否需要手动创建页面 | 使用模板 | 推荐说明 |
| --- | --- | --- | --- |
| 首页 | 需要 | `首页` | 创建后到 `设置 -> 阅读` 设为静态首页 |
| 关于我们 | 需要 | `关于我们` | 推荐别名 `about-us` |
| 点点滴滴 | 需要 | `点点滴滴` | 推荐别名 `moments` |
| 甜蜜相册 | 需要 | `甜蜜相册` | 推荐别名 `memories` |
| 随笔说说 | 需要 | `随笔说说` | 推荐别名 `notes` |
| 祝福留言 | 需要 | `祝福留言` | 页面评论需要保持开启 |
| 恋爱清单 | 不需要 | 无 | 这是 `love_list` 的归档页，默认链接为 `/lists/` |

## 页面功能与使用方法

### 1. 首页

用途：作为站点入口页，集中展示时间、天气、纪念日和各模块跳转入口。

如何启用：
1. 新建一个普通页面
2. 右侧模板选择 `首页`
3. 到 `设置 -> 阅读` 将它设置为静态首页

你需要维护的内容：
- `外观 -> 自定义 -> Brave 主题设置 -> 基本信息`
  - 恋爱起始时间
  - 首页正计时文案
  - 下一个纪念日倒计时名称与时间
- `设置 -> 天气城市`
  - 控制首页天气模块是否显示
  - 支持添加任意数量城市，不限制 4 个
  - 支持在后台拖拽调整首页天气卡片顺序
- `设置 -> 纪念日`
  - 控制“特别的日子”卡片内容

前台表现说明：
- `恋爱时长` 会根据恋爱起始时间实时跳动
- `下一个纪念日` 倒计时会精确到秒
- `今日天气` 点击卡片会打开详情弹层，显示温度、空气质量、紫外线、穿搭建议与 6 小时趋势
- `特别的日子` 卡片按“离今天最近优先”排序，不按添加顺序排序
- 下方六张入口卡片会自动跳到站点各个核心页面

### 2. 关于我们

用途：展示完整的故事时间线，适合写“我们怎么认识、怎么走到一起、经历了什么”。

如何启用：
1. 新建一个页面
2. 模板选择 `关于我们`
3. 可在页面右侧 `页面 Hero 设置` 中填写 Hero 标题 / 副标题 / 背景图

内容如何维护：
- 页面正文内容：作为顶部“写在前面”区域
- 时间线内容：通过后台左侧菜单 `关于我们` 中新增“故事节点”来维护

每个故事节点建议填写：
- 标题：节点标题，如“第一次见面”
- 摘要：使用 WordPress 原生 `摘要` 面板填写
- 正文：该节点的完整故事内容
- 特色图：节点插图
- 节点日期：控制时间线顺序
- 阶段标题：例如“相遇 / 靠近 / 稳定 / 未来”
- 关联点点滴滴：可选，前台会出现跳转入口

前台表现说明：
- 故事节点按 `_story_date` 正序排列，最早的在前面
- 如果摘要为空，卡片中不会显示摘要
- 如果设置了关联点滴，节点底部会出现“关联点点滴滴”链接
- 页面顶部会根据节点总数、首个节点日期、最新节点自动生成概览信息

### 3. 点点滴滴

用途：发布更完整的记录文章，是整站最核心的数据来源之一。

如何启用：
1. 新建一个页面
2. 模板选择 `点点滴滴`
3. 建议到 `外观 -> 自定义 -> Brave 主题设置 -> 页面链接` 中指定这张页面

如何添加内容：
- 左侧菜单进入 `点点滴滴 -> 添加点滴`
- 每篇点滴建议填写：
  - 标题
  - 正文
  - 原生摘要 `Excerpt`
  - 特色图
  - 见面日期
  - 见面地点
  - 心情

摘要规则：
- 前台优先使用 WordPress 原生摘要 `post_excerpt`
- 如果原生摘要为空：
  - 列表页卡片会自动截取正文前 `120` 个词作为简介
  - 详情页不会单独显示摘要块

前台表现说明：
- 列表按 `见面日期` 倒序显示，新的在前面
- 可按年份筛选
- 作者头像会优先复用主题里设置的情侣头像
- 上传到正文或设置为特色图的图片，会自动进入 `甜蜜相册`

### 4. 恋爱清单

用途：记录“想一起完成的事情”和“已经完成的事情”。

如何启用：
- 不需要创建页面
- 这是一个文章类型归档页，默认访问路径为 `/lists/`
- 首页入口和页脚导航会自动指向这个归档页

如何添加内容：
- 左侧菜单进入 `恋爱清单 -> 添加事项`
- 可选设置 `清单分类`
- 每个事项建议填写：
  - 标题
  - 正文
  - 特色图
  - 是否已完成
  - 完成日期

前台表现说明：
- 顶部会自动统计：已完成、待完成、完成度
- 支持按分类筛选
- 支持按状态筛选：`已完成 / 待完成`
- 默认排序为：`待完成在前`，`已完成` 按完成日期从近到远排列
- 已完成事项如果填写了正文或特色图，卡片支持展开查看详情
- 未完成事项只显示简洁卡片信息
- 后台 `恋爱清单` 管理列表默认也会沿用同样的排序逻辑，方便维护

补充说明：
- `恋爱清单` 的 Hero 不走页面模板，而是在 `外观 -> 自定义 -> Brave 主题设置 -> Hero 区域` 里单独设置

### 5. 甜蜜相册

用途：集中浏览你们所有照片，不需要重复录入图片数据。

如何启用：
1. 新建一个页面
2. 模板选择 `甜蜜相册`
3. 建议在 `页面链接` 中指定这张页面

它的数据来源：
- `点点滴滴` 的特色图
- `点点滴滴` 正文里的图片

前台表现说明：
- 自动按年份筛选
- 瀑布流布局显示
- 点击图片会进入 PhotoSwipe 灯箱
- 右侧信息面板可展示日期、地点、心情、摘要与原始点滴链接

可配置项：
- `外观 -> 自定义 -> Brave 主题设置 -> 相册设置`
  - 每页照片数
  - 是否显示悬浮信息

注意：
- 新站点通常不需要手动维护这个页面的数据
- 只要持续发布 `点点滴滴` 并上传图片，相册会自动更新

### 6. 随笔说说

用途：发布轻量、碎片化的日常短内容。

如何启用：
1. 新建一个页面
2. 模板选择 `随笔说说`
3. 建议在 `页面链接` 中指定这张页面

有两种发布方式：
- 后台发布：`随笔说说 -> 添加说说`
- 前台快捷发布：登录后直接在页面底部表单发布

后台可维护字段：
- 正文
- 心情表情
- 思念度（1-5 星）

前台表现说明：
- 按发布时间倒序显示
- 支持 `年份 -> 月份 -> 日期` 三级筛选
- 登录且有发布权限的用户，可以在前台直接发布说说
- 作者头像优先使用主题中配置的情侣头像

### 7. 祝福留言

用途：让亲友或访客给你们留言祝福。

如何启用：
1. 新建一个页面
2. 模板选择 `祝福留言`
3. 确保该页面 `评论` 是开启状态

你需要知道的规则：
- 访客通过页面底部表单提交昵称、邮箱和祝福内容
- 所有新留言默认进入审核后才显示
- 前台只显示 `已审核` 留言

前台表现说明：
- 留言按时间倒序显示
- 支持按年份筛选
- 使用主题内置头像池，不依赖外部头像服务
- 如果页面关闭评论，底部表单会自动隐藏

## 后台入口说明

### 外观 -> 自定义 -> Brave 主题设置

这里是主题的总控台，重点配置项如下：

- `基本信息`
  - 恋爱起始时间
  - 首页正计时文案
  - 下一个纪念日名称与时间
  - 导航副标题等基础信息
- `纪念日`
  - 首页“特别的日子”标题
  - 跳转到纪念日管理页
- `Hero 区域`
  - 全局 Hero 背景
  - 情侣头像与昵称
  - 情侣头像可绑定 WordPress 用户 ID
  - 恋爱清单归档 Hero 文案与背景图
- `入口图标`
  - 首页入口卡片 Emoji 图标
- `页面链接`
  - 指定关于我们 / 点点滴滴 / 相册 / 说说 / 祝福留言页面
- `页脚导航`
  - 单独自定义页脚名称和链接
- `分页设置`
  - 点点滴滴每页文章数
- `相册设置`
  - 相册每页照片数
  - 是否显示图片信息
- `自定义代码`
  - 自定义 CSS
  - 页脚统计代码
- `访问统计`
  - 今日 / 累计访问文案
  - 手动覆盖访问数值

### 设置 -> 天气城市

作用：维护首页天气模块。

可做的事：
- 开关天气模块
- 添加任意数量城市
- 为每个城市填写名称、纬度、经度
- 通过拖拽左侧手柄调整首页天气卡片顺序

填写建议：
- 坐标格式使用十进制度，例如 `31.2304` / `121.4737`
- 主题天气数据使用 QWeather API Key，需要在服务端额外配置 `QWEATHER_API_HOST` 和 `QWEATHER_API_KEY`
- 如果天气不显示，优先检查坐标是否填写正确

### 设置 -> 纪念日

作用：维护首页“特别的日子”卡片。

可做的事：
- 添加纪念日名称
- 选择纪念日日期
- 删除或调整纪念日条目

排序说明：
- 前台不是按添加顺序显示
- 而是按“离今天最近优先”排序

### 点点滴滴 -> 相册数据管理

仅旧站迁移用户需要关注。

作用：处理旧版 `memory` 数据。

支持：
- 将旧 `memory` 转换为 `moment`
- 删除旧相册数据

如果你是新装站点，可以忽略这个页面。

### 点点滴滴 -> 摘要迁移

仅曾经使用旧版 `_moment_summary` 自定义摘要字段的站点需要关注。

作用：把旧摘要迁移到 WordPress 原生摘要 `post_excerpt`。

推荐流程：
1. 先点击“迁移到原生摘要”
2. 确认前台显示正常
3. 再点击“清理已迁移旧字段”

新装站点通常不需要执行这个页面的操作。

## 内容模型与字段一览

| 内容类型 | 主要字段 | 用途 |
| --- | --- | --- |
| `moment` | 标题、正文、摘要、特色图、见面日期、地点、心情 | 点点滴滴主内容，同时为相册提供图片数据 |
| `story_milestone` | 标题、摘要、正文、特色图、节点日期、阶段标题、关联点滴 | 关于我们时间线节点 |
| `love_list` | 标题、正文、特色图、是否已完成、完成日期、分类 | 恋爱清单事项 |
| `note` | 正文、作者、心情表情、思念度 | 随笔说说 |
| `page` Hero 元信息 | Hero 标题、Hero 副标题、Hero 背景图 | 用于关于我们 / 点点滴滴 / 相册 / 说说 / 留言等页面头图 |

## 常见操作说明

### 如何修改页面头图和文案？
- 普通页面：进入对应页面编辑页，使用右侧 `页面 Hero 设置`
- 恋爱清单：进入 `外观 -> 自定义 -> Brave 主题设置 -> Hero 区域`

### 如何让“关于我们”显示摘要？
- 在故事节点编辑器中打开并填写 WordPress 原生 `摘要` 面板
- 如果摘要为空，前台不会显示摘要

### 如何让“点点滴滴”显示摘要？
- 同样使用编辑器自带的 `摘要` 面板
- 如果摘要为空，列表页会自动截取正文，详情页不额外显示摘要块

### 为什么我上传的图片出现在甜蜜相册里？
- 因为相册会自动聚合 `点点滴滴` 的特色图和正文图片
- 这是设计如此，不需要再单独上传一次

### 为什么祝福留言页看不到表单？
- 请检查该页面是否开启评论
- 也请检查 `设置 -> 讨论` 是否阻止了评论功能

### 为什么首页天气没有显示？
- 检查 `设置 -> 天气城市` 是否已启用天气模块
- 检查城市是否填写了正确的经纬度
- 检查 `wp-config.php` 或环境变量里是否已配置 `QWEATHER_API_HOST` 与 `QWEATHER_API_KEY`
- 检查首页是否确实使用了 `首页` 模板

## 推荐初始化顺序

如果你是第一次搭站，推荐按下面顺序配置：

1. 启用主题
2. 创建 6 张页面并分配模板
3. 设置首页为静态首页
4. 在 `自定义` 中配置头像、基础信息、页面链接
5. 添加天气城市与纪念日
6. 先录入几篇 `点点滴滴`
7. 再录入 `关于我们` 的故事节点
8. 最后补充 `恋爱清单`、`随笔说说` 与 `祝福留言`

这样可以最快看到一个完整可用的网站雏形。

## 版本说明

当前 `v1.0.8` 已包含：

- 首页天气已切换到 QWeather API Key 方案，详情弹窗支持空气质量、空气预报、贴心提醒与今日降雨概率
- 首页天气城市后台支持拖拽排序，保存后前台按拖拽顺序展示
- 恋爱清单前台支持按完成状态与完成日期自动排序，后台管理列表默认同步相同逻辑
- 本次 patch 额外清理了旧分钟降雨链路，并把天气请求坐标精度恢复到 4 位小数

如果你推送新的 tag，仓库中的 GitHub Actions 发布流程会自动构建 Release 资产。
