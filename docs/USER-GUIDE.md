# Brave Love 用户使用手册

这是一份给站长本人看的操作手册，目标不是解释主题源码，而是告诉你：这个站怎么搭、每个页面怎么用、平时该去哪里维护内容。

如果你刚装好主题，建议先看：

1. `30 分钟快速上线`
2. `六大页面怎么建`
3. `后台入口速查`
4. `日常维护工作流`

## 主题适合怎么用

Brave Love 适合做一个长期更新的情侣纪念站，而不是一次性静态页。

你可以把它理解成 6 个内容模块加 1 个首页入口：

- `首页`：展示时间、天气、特别日子和所有入口
- `关于我们`：一条完整的故事线
- `点点滴滴`：一篇篇正式记录
- `恋爱清单`：想做的事、已经完成的事
- `甜蜜相册`：自动汇总照片
- `随笔说说`：轻量碎片记录
- `祝福留言`：亲友留言板

最核心的数据来源其实是 `点点滴滴`。很多内容都会和它联动，尤其是 `甜蜜相册` 和 `关于我们` 的关联跳转。

## 30 分钟快速上线

### 第一步：启用主题
1. 后台进入 `外观 -> 主题`
2. 启用 `Brave Love`
3. 进入 `设置 -> 固定链接`
4. 不用改任何内容，直接点击一次“保存更改”

### 第二步：创建 6 张页面
请手动创建以下页面，并为它们选择对应模板：

| 页面名 | 模板 |
| --- | --- |
| 首页 | `首页` |
| 关于我们 | `关于我们` |
| 点点滴滴 | `点点滴滴` |
| 甜蜜相册 | `甜蜜相册` |
| 随笔说说 | `随笔说说` |
| 祝福留言 | `祝福留言` |

说明：
- `恋爱清单` 不需要手动建页面，它本身就是归档页
- 建议 `关于我们` 的别名使用 `about-us`
- 建议 `点点滴滴 / 甜蜜相册 / 随笔说说` 的别名分别使用 `moments / memories / notes`

### 第三步：设置首页
1. 后台进入 `设置 -> 阅读`
2. 首页显示选择 `一个静态页面`
3. 首页选择你刚才创建的“首页”页面

### 第四步：进入主题设置总控台
路径：`外观 -> 自定义 -> Brave 主题设置`

优先配置这些内容：
- 恋爱起始时间
- 首页正计时文案
- 下一个倒计时名称和时间
- 情侣头像
- 情侣昵称
- 页面绑定与跳转
- 外观 -> 自定义 -> Brave 主题设置 -> 页脚导航

### 第五步：补齐首页数据
继续完成：
- `设置 -> 天气城市`：添加想展示的城市，并拖拽调整首页显示顺序
- `设置 -> 纪念日`：添加特别的日子

### 第六步：开始填内容
推荐顺序：
1. 先写几篇 `点点滴滴`
2. 再补 `关于我们` 的故事节点
3. 然后补 `恋爱清单`
4. 最后再补 `随笔说说` 和 `祝福留言`

这样首页会很快丰满起来。

## 六大页面怎么建

### 首页

作用：
- 展示恋爱时长
- 展示下一个纪念日倒计时
- 展示天气
- 展示特别的日子
- 提供所有主要入口

你要去哪里维护：首页本身几乎不直接写内容，主要去下面几个地方：
- `外观 -> 自定义 -> Brave 主题设置 -> 首页与情侣`
- `外观 -> 自定义 -> Brave 主题设置 -> 纪念日与倒计时`
- `设置 -> 天气城市`
- `设置 -> 纪念日`

特别说明：
- `今日天气` 支持添加任意数量城市
- `今日天气` 的城市顺序支持后台拖拽手动调整
- `特别的日子` 不是按添加顺序排，而是按“离今天最近优先”排序

### 关于我们

作用：
- 用故事节点拼出一条时间线
- 页面正文负责“写在前面”
- 节点负责具体故事

页面本身怎么写：
- 页面正文：写前言、总述、开场白
- Hero：在页面右侧 `当前页面 Hero` 里设标题、副标题、背景图

故事节点怎么写：
后台菜单：`关于我们`

每个节点建议填写：
- 标题
- 原生摘要
- 正文
- 特色图
- 节点日期
- 阶段标题
- 关联点点滴滴（可选）

注意：
- 节点按日期正序排，越早越前
- 摘要为空时，前台不会自动截正文
- 如果关联了某篇点滴，前台会自动给出跳转入口

### 点点滴滴

作用：
- 记录正式内容
- 作为相册数据来源
- 可以被关于我们节点关联

后台菜单：`点点滴滴`

每篇建议填写：
- 标题
- 正文
- 原生摘要
- 特色图
- 见面日期
- 见面地点
- 心情

摘要规则：
- 有摘要：优先显示摘要
- 没摘要：列表自动截正文，详情页不单独显示摘要区块

日期规则：
- 优先用 `见面日期`
- 如果没填，回退文章发布日期

### 恋爱清单

作用：
- 管理待完成和已完成事项
- 适合做“我们想一起完成的 100 件事”这类页面

后台菜单：`恋爱清单`

每个事项建议填写：
- 标题
- 正文
- 特色图
- 是否已完成
- 完成日期
- 清单分类（可选）

前台特点：
- 自动统计完成度
- 支持按分类筛选
- 支持按完成状态筛选
- 默认排序为：待完成在前，已完成按完成日期从近到远排列
- 只有已完成且有内容/图片的事项才会展开详情
- 后台 `恋爱清单` 列表默认也按相同逻辑排序，方便你快速看到还没完成的事项

### 甜蜜相册

作用：
- 自动汇总照片
- 不用重复建图集

数据来源：
- 点点滴滴的特色图
- 点点滴滴正文里的图片

所以你日常维护时要记住：
- 不是去“相册页面”上传图
- 而是去写 `点点滴滴`，图片自然会汇总过来

相册相关设置位置：
`外观 -> 自定义 -> Brave 主题设置 -> 列表与相册展示`

你可以配置：
- 每页多少张照片
- 是否显示信息覆盖层

### 随笔说说

作用：
- 适合发短句、心情、念想
- 比点点滴滴更轻

后台菜单：`随笔说说`

两种发布方式：
- 后台发布
- 前台登录后快捷发布

建议写法：
- 正文尽量短
- 配合心情表情
- 思念度 1-5 星按当天感觉填

适合写什么：
- 想你了
- 今天发生的小事
- 某句话想记下来
- 某个瞬间的情绪

### 祝福留言

作用：
- 给朋友、家人、访客留言
- 适合在纪念日或结婚站点开放

注意事项：
- 页面必须开启评论
- 留言默认审核后才显示
- 前台只显示已审核的评论

你的日常工作：
- 去后台审核评论
- 删除垃圾留言
- 保留真正的祝福内容

## 后台入口速查

### 外观 -> 自定义 -> Brave 主题设置
这是最重要的主题总控台。

建议按下面的逻辑理解：
- `首页与情侣`：改首页恋爱正计时、首页顶部导航副标题、情侣头像昵称；作者用户 ID 只给 `点点滴滴 / 随笔说说` 的作者样式映射用
- `默认页面 Hero`：只放内容页默认 Hero 背景；某一页想单独改标题、副标题或背景，请去页面编辑器右侧 `当前页面 Hero`
- `纪念日与倒计时`：只管首页顶部倒计时和“特别的日子”模块标题，真正的纪念日列表在 `设置 -> 纪念日`
- `页面绑定与跳转`：把关于我们、点点滴滴、相册、说说、留言、旅行计划这些普通页面绑定给主题
- `首页入口卡片图标`：只改首页入口卡片的 Emoji 图标
- `恋爱清单页（归档）`：单独改恋爱清单归档页的 Hero
- `列表与相册展示`：控制点点滴滴每页数量、相册每页数量和相册信息显示
- `页脚导航`：拖拽排序、改单项名称和链接；链接留空时自动继承页面绑定
- `页脚访客统计`：改今日 / 累计访客文案；手动覆盖数字只在校正数据时使用
- `自定义代码`：放自定义 CSS 和页脚统计 / 验证代码

### 设置 -> 天气城市
这里维护首页天气模块。

填写规则：
- 城市名：随便写你想显示的名称
- 纬度 / 经度：必须准确
- 支持添加任意数量城市
- 支持拖拽左侧手柄排序，保存后首页会按这个顺序展示
- 可以直接在这里填写 `QWeather API Host` 和 `QWeather API Key`
- `API Host` 就是和风天气给你的接口域名，例如 `nb7aarhnan.re.qweatherapi.com`
- 如果服务器环境中也配置了同名常量或环境变量，服务器配置优先，后台保存值作为兜底

适合的维护方式：
- 保留你们常驻城市
- 再加上异地城市、老家或旅行地

### 设置 -> 纪念日
这里维护首页特别的日子卡片。

适合添加：
- 恋爱纪念日
- 生日
- 第一次见面
- 领证日
- 婚礼日
- 旅行出发日

显示规则：
- 离今天最近的会优先显示
- 未来日子排在前，过去的排后

### 点点滴滴 -> 相册数据管理
这个页面只给旧站迁移用。

如果你是新站：
- 可以直接忽略

如果你是老版本升级：
- 这里可以处理旧的 `memory` 数据
- 可以转换成 `moment`
- 也可以删除旧数据

### 点点滴滴 -> 摘要迁移
这个页面也只给旧站迁移用。

如果你以前使用过旧的 `_moment_summary` 字段：
1. 先迁移到原生摘要
2. 再检查前台
3. 最后清理旧字段

如果你现在是新写内容：
- 直接用 WordPress 编辑器的原生 `摘要` 面板
- 不需要再管迁移工具

## 日常维护工作流

### 工作流 1：更新一篇正式记录
适用场景：旅行、纪念日、约会、重要节点。

步骤：
1. 新建一篇 `点点滴滴`
2. 填标题、正文、摘要
3. 上传特色图
4. 补见面日期、地点、心情
5. 发布
6. 检查：
   - 点点滴滴页是否正常显示
   - 甜蜜相册是否自动带出图片

### 工作流 2：补一段故事线
适用场景：想把旧回忆整理进“关于我们”。

步骤：
1. 去 `关于我们` 新增故事节点
2. 填节点日期、标题、摘要、正文
3. 可选关联一篇已有点滴
4. 发布
5. 去前台确认时间线顺序和摘要显示

### 工作流 3：更新首页纪念日
适用场景：新增一个纪念日或想调整显示内容。

步骤：
1. 去 `设置 -> 纪念日`
2. 新增或修改日期
3. 保存
4. 回首页看排序是否符合预期

### 工作流 4：发布一句短内容
适用场景：今天突然很想写一句话。

步骤：
- 后台：去 `随笔说说 -> 添加说说`
- 或前台登录后直接快捷发布

建议：
- 说说适合简短一点
- 长内容还是交给 `点点滴滴`

### 工作流 5：维护访客留言
适用场景：站点开放后有人留言。

步骤：
1. 后台进入评论管理
2. 审核真实留言
3. 删除垃圾评论
4. 必要时关闭页面评论

## 常见问题

### 为什么首页没有显示天气？
检查这几项：
- 首页是不是用了 `首页` 模板
- `设置 -> 天气城市` 是否已开启
- 是否至少填写了一条有效城市坐标

### 为什么关于我们不显示摘要？
- 因为没填写原生摘要
- 现在逻辑是：有摘要才显示，没摘要就不显示

### 为什么点点滴滴有时显示正文截取？
- 因为那篇文章没有填写原生摘要
- 列表页会自动截正文作为简介

### 为什么甜蜜相册里有图，但我没单独上传到相册页？
- 相册会自动抓取点点滴滴的图片
- 这是正常设计

### 为什么祝福留言页没有提交表单？
- 先检查页面评论是否开启
- 再检查 WordPress 讨论设置

### 为什么纪念日顺序和我添加顺序不一样？
- 主题按离今天最近优先排序
- 不是按你后台录入顺序显示

## 内容维护建议

为了前台效果稳定，建议你遵循下面的习惯：

- `点点滴滴`：尽量每篇都填摘要、日期、地点、特色图
- `关于我们`：每个故事节点尽量都填摘要和节点日期
- `恋爱清单`：完成后及时勾选“已完成”并补完成日期
- `随笔说说`：适合短、轻、快，不要和点滴重复职责
- `祝福留言`：定期审核，避免垃圾评论堆积

## 推荐内容分工

如果你想让整个站更有层次，推荐这样分工：

- `关于我们`：写主线故事
- `点点滴滴`：写每次重要记录
- `随笔说说`：写短心情
- `恋爱清单`：写计划和完成感
- `甜蜜相册`：只负责展示，不单独维护数据
- `祝福留言`：只负责对外互动

## 最后建议

如果你准备长期运营这个站，最值得坚持的不是每天折腾样式，而是形成内容习惯：

- 每次有正式回忆，就写一篇 `点点滴滴`
- 每隔一段时间，整理一条 `关于我们` 节点
- 想说一句话时，就发到 `随笔说说`
- 做完一件想一起完成的事，马上去勾掉 `恋爱清单`

这样一年以后，这个站会非常完整。
